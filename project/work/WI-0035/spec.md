## Goal

Hard-migrate the Astro blog to mainstream canonical URL conventions:

- HTML pages use extensionless URLs such as `/posts/e4dcbcaf`, `/archives`, `/categories/music`, and `/page/2`.
- XML resources keep explicit `.xml`, including RSS/Atom and sitemap files.
- RSS/feed and sitemap agree with the canonical page URL shape.

## Scope

1. Change Astro output/routing so generated HTML pages use directory-style extensionless canonical URLs.
2. Remove `.html` from internal links generated by pages, theme components, post navigation, archives, categories, tags, pagination, search index, and copyright URLs.
3. Update feed item URLs to extensionless post links while keeping the XML feed endpoint.
4. Keep sitemap generated by Astro with XML files and extensionless page URLs.
5. Update Fluid Astro theme config/integration route defaults and README route table to match the new canonical shape.
6. Keep external URLs inside old authored Markdown content out of scope unless they are generated by current templates/config.

## Non-goals

- Do not add redirect compatibility for old `.html` URLs. The owner requested a hard migration.
- Do not redesign pages.
- Do not edit generated `dist/**` directly.
- Do not change post `abbrlink` values.
- Do not rename `atom.xml` unless required for consistency.

## Acceptance Criteria

1. `npm run build` passes.
2. Generated post pages are written as `dist/posts/<abbrlink>/index.html`, not `dist/posts/<abbrlink>.html`.
3. Rendered internal links in generated pages do not contain `.html` for site pages.
4. `dist/atom.xml` item links use extensionless post URLs.
5. `dist/sitemap-index.xml` and `dist/sitemap-0.xml` exist and sitemap page URLs are extensionless.
6. Search index URLs are extensionless.

## Risks

- Hard migration breaks old inbound `.html` links. This is accepted by the owner request.
- Theme route injection and host `src/pages` can drift. Mitigation: update both copies.
